欢迎使用 SQLHunter

SQLHunter 是一款基于 sqlmap 的现代化 SQL 注入检测工具。它提供直观的图形界面， 集成智能代理池、Tor 匿名网络和 AI 智能分析功能，让 SQL 注入检测变得简单而强大。 本文档将帮助你快速上手并充分利用所有功能。

核心功能

智能扫描引擎

支持 Boolean、Error、Union、Time-based 等多种注入技术，自动识别数据库类型和后端环境

智能代理池

自动获取和管理代理，支持自动轮换和健康检查，内置 Tor 匿名网络集成

可视化配置

直观的参数配置界面，内置多种扫描预设，告别复杂的命令行参数

AI 结果分析

智能分析扫描结果，可视化数据库结构树，提供漏洞评估和修复建议

环境检测

首次启动时，应用会自动检测依赖环境。打开设置面板可以查看：

  • Python 状态：显示 Python 版本和安装路径
  • sqlmap 状态：显示 sqlmap 版本，如未安装可点击「一键修复」
  • Tor 状态：显示 Tor 是否已安装和运行状态

提示：如果 sqlmap 未安装，点击环境检测中的「一键修复」按钮，程序会自动通过 pip 安装 sqlmap。

快速开始

按照以下三个步骤完成你的第一次 SQL 注入检测：

第一步：输入目标 URL

在应用顶部的输入框中填入包含可能存在注入点的目标 URL。URL 应该包含查询参数，例如：

http://example.com/page.php?id=1

sqlmap 会自动识别 id 参数并尝试进行注入测试。

第二步：选择扫描配置

在左侧配置面板中选择扫描参数。对于初学者，建议使用内置预设：

  • 快速扫描：使用基本技术快速检测，适合初步测试
  • 深度扫描：启用所有技术，更高的检测级别和风险级别
  • WAF 绕过：针对有 WAF 防护的目标，启用 tamper 脚本

第三步：开始扫描

点击「开始扫描」按钮或使用快捷键 Ctrl/Cmd + Enter 启动扫描。扫描过程中：

  • 中央终端区域会显示 sqlmap 的实时输出
  • 右侧会显示发现的数据库结构
  • 扫描完成后可查看详细结果和报告

注意：请确保你有权限对目标进行安全测试。未经授权的测试是违法行为。

目标配置

SQLHunter 支持多种目标输入方式，满足不同的测试场景。

URL 目标

最简单的方式是直接输入包含参数的 URL。sqlmap 会自动识别 URL 中的参数作为注入点：

http://target.com/page.php?id=1&name=test

POST 数据

对于 POST 请求，你可以在配置面板中填写 POST 数据：

username=admin&password=123456

完整 HTTP 请求

支持粘贴 Burp Suite、Fiddler 等工具截获的完整 HTTP 请求：

POST /login.php HTTP/1.1 Host: target.com Content-Type: application/x-www-form-urlencoded Cookie: session=abc123 username=admin&password=123456

自定义注入点

使用 * 标记指定注入点位置：

http://target.com/page.php?id=1*&name=test

Cookie 注入

在配置面板的 Cookie 字段中填入要测试的 Cookie：

session=abc123; user_id=1

注入技术

SQLHunter 支持 sqlmap 的所有 SQL 注入检测技术。你可以在配置面板中通过复选框选择要使用的技术。

代码 技术名称 说明
B Boolean-based blind 基于布尔值的盲注。通过比较页面响应差异判断条件真假。适用于大多数情况。
E Error-based 基于错误信息的注入。利用数据库错误信息获取数据。效率高但需要错误回显。
U Union-based 基于 UNION 查询的注入。通过 UNION 合并查询获取数据。效率最高。
S Stacked queries 堆叠查询注入。可以执行多条 SQL 语句。支持写入文件、执行命令等高级操作。
T Time-based blind 基于时间延迟的盲注。通过响应时间判断条件真假。最慢但最可靠。
Q Inline queries 内联查询注入。将查询嵌入到原始查询中。

检测级别 (Level)

检测级别控制测试的范围和深度。级别越高，测试的参数和位置越多：

  • Level 1：只测试 GET 和 POST 参数（默认）
  • Level 2：额外测试 Cookie
  • Level 3：额外测试 User-Agent 和 Referer
  • Level 4：测试更多的边界情况
  • Level 5：最全面的测试，包括更多的 payload 变体

风险级别 (Risk)

风险级别控制允许使用的 payload 类型：

  • Risk 1：只使用无害的测试 payload（默认）
  • Risk 2：额外使用重查询时间的 payload
  • Risk 3：额外使用基于 OR 的 payload（可能影响数据）

扫描预设

SQLHunter 内置多种扫描预设，帮助你快速配置常用的扫描策略。

内置预设

  • 快速扫描
    Level: 1, Risk: 1, 技术: BEUST
    适合初步检测，速度快，误报少
  • 深度扫描
    Level: 5, Risk: 3, 技术: BEUSTQ
    全面检测，不遗漏任何可能的漏洞
  • WAF 绕过
    启用 tamper 脚本，随机 User-Agent
    针对有 WAF 防护的目标
  • 仅检测
    只检测是否存在漏洞，不进行数据提取
    适合快速评估

自定义预设

你可以保存当前配置为自定义预设：

  1. 在配置面板中调整好所有参数
  2. 点击预设下拉菜单旁边的「保存」按钮
  3. 输入预设名称并确认

保存的预设会显示在预设列表中，下次可以一键加载。

结果分析

扫描完成后，SQLHunter 提供多种方式查看和分析结果。

实时输出

中央终端区域显示 sqlmap 的完整输出，包括：

  • 测试进度和状态
  • 发现的注入点详情
  • 数据库类型和版本
  • 提取的数据内容

数据库结构树

右侧面板以树形结构展示发现的数据库内容：

  • 数据库列表
  • 每个数据库下的表
  • 每个表的列名和类型

AI 智能分析

配置 API Key 后，可以使用 AI 分析功能：

  • 自动分析扫描结果
  • 评估漏洞的严重程度
  • 提供修复建议
  • 生成安全报告

智能代理池

智能代理池可以自动管理代理，避免在扫描过程中被目标封禁 IP。点击右上角的代理池图标打开管理面板。

主要功能

  • 代理获取：从配置的代理源自动获取代理
  • 健康检查：定期检测代理可用性，自动剔除失效代理
  • 自动轮换：每 N 次请求自动切换代理
  • 手动添加：支持手动添加自己的代理

使用方法

  1. 打开代理池面板
  2. 在「代理源」标签页添加代理获取源
  3. 点击「获取代理」按钮获取代理列表
  4. 启用「智能代理池」开关
  5. 开始扫描，代理会自动应用

代理源管理

代理源是提供代理列表的 URL 地址。SQLHunter 支持添加多个代理源。

添加代理源

  1. 打开代理池面板，切换到「代理源」标签
  2. 点击「添加」按钮
  3. 填写代理源名称和 URL
  4. URL 应返回每行一个代理的文本格式

代理格式

代理源返回的代理格式应为：

192.168.1.1:8080 192.168.1.2:3128 socks5://192.168.1.3:1080

Tor 网络

SQLHunter 集成了 Tor 匿名网络，可以一键启用 Tor 代理进行匿名扫描。

安装 Tor

使用 Tor 功能前，需要先安装 Tor：

  • macOS：brew install tor
  • Linux：sudo apt install tor
  • Windows：下载安装 Tor Expert Bundle

使用 Tor

  1. 打开代理池面板，切换到「Tor」标签
  2. 点击「启动 Tor」按钮
  3. 等待 Tor 连接成功，显示出口 IP
  4. 开始扫描，流量将通过 Tor 网络

Tor 链路可视化

Tor 标签页会实时显示当前的 Tor 链路信息：

  • 入口节点及其国家
  • 中继节点及其国家
  • 出口节点及其国家
  • 当前出口 IP 地址

切换身份

点击「切换身份」按钮可以获取新的 Tor 出口 IP，用于：

  • 更换被封禁的 IP
  • 获取不同地理位置的出口

代理设置

在代理池的「设置」标签页可以配置代理相关参数。

设置项 说明
启用代理池 开启后扫描时会自动使用代理
自动轮换 每 N 次请求自动切换到下一个可用代理
自动验活 每 N 秒自动检查代理可用性
最大失败次数 代理连续失败该次数后标记为不可用
代理超时 代理请求的超时时间（毫秒）

任务队列

任务队列允许你批量添加多个扫描任务，并自动按顺序执行。

添加任务

  1. 配置好目标和扫描参数
  2. 点击「添加到队列」按钮
  3. 任务会添加到右侧的任务列表

任务管理

  • 开始队列：按顺序执行所有待处理任务
  • 暂停队列：暂停当前任务执行
  • 删除任务：从队列中移除任务
  • 重试任务：重新执行失败的任务

目标管理器

目标管理器帮助你组织和管理多个测试目标。

功能

  • 添加目标：保存目标 URL 和相关信息
  • 分组管理：将目标按项目或类型分组
  • 标签标记：使用标签对目标进行分类
  • 批量导入：从文件导入多个目标
  • 快速扫描：一键对目标发起扫描

AI 分析

SQLHunter 支持集成 AI 大模型来分析扫描结果。

配置 API

  1. 打开设置面板
  2. 在 AI 设置中选择服务提供商
  3. 填入 API Key
  4. 选择要使用的模型

支持的服务

  • OpenAI（GPT-4、GPT-3.5）

  • 自定义 API 端点

使用 AI 分析

扫描完成后，点击「AI 分析」按钮，AI 会：

  • 分析扫描输出中的关键信息
  • 评估漏洞的严重程度
  • 提供详细的漏洞说明
  • 给出修复建议
