Welcome to SQLHunter

SQLHunter is a modern SQL injection detection tool based on sqlmap. It provides an intuitive graphical interface with smart proxy pool, Tor anonymity network, and AI-powered analysis. This documentation will help you get started quickly.

Core Features

Smart Scanning Engine

Supports Boolean, Error, Union, Time-based and more injection techniques with auto database detection

Smart Proxy Pool

Auto-fetch and manage proxies with health checks and rotation. Built-in Tor integration

Visual Configuration

Intuitive parameter settings with built-in scan presets. No command-line knowledge required

AI Result Analysis

Smart scan result analysis with visual database structure and vulnerability assessment

Environment Check

On first launch, the app will check dependencies. Open Settings to view:

  • Python Status: Shows Python version and path
  • sqlmap Status: Shows sqlmap version, click "Auto Fix" if missing
  • Tor Status: Shows Tor installation and running status

Tip: If sqlmap is not installed, click "Auto Fix" in Environment Check to install via pip automatically.

Quick Start

Complete your first SQL injection scan in 3 steps:

Step 1: Enter Target URL

Enter a URL with potential injection points in the input field:

http://example.com/page.php?id=1

sqlmap will automatically detect the id parameter for injection testing.

Step 2: Choose Scan Configuration

Select scan parameters in the left panel. For beginners, use built-in presets:

  • Quick Scan: Fast detection with basic techniques
  • Deep Scan: Comprehensive testing with all techniques
  • WAF Bypass: For targets with WAF protection, enables tamper scripts

Step 3: Start Scan

Click "Start Scan" or press Ctrl/Cmd + Enter. During scanning:

  • Center terminal shows real-time sqlmap output
  • Right panel displays discovered database structure
  • View detailed results after completion

Warning: Ensure you have authorization to test the target. Unauthorized testing is illegal.

Target Configuration

SQLHunter supports multiple target input methods:

URL Target

Enter a URL with parameters. sqlmap auto-detects parameters as injection points:

http://target.com/page.php?id=1&name=test

POST Data

For POST requests, fill in POST data in the config panel:

username=admin&password=123456

Full HTTP Request

Paste complete HTTP requests from Burp Suite, Fiddler, etc:

POST /login.php HTTP/1.1 Host: target.com Content-Type: application/x-www-form-urlencoded Cookie: session=abc123 username=admin&password=123456

Custom Injection Point

Use * marker to specify injection points:

http://target.com/page.php?id=1*&name=test

Injection Techniques

SQLHunter supports all sqlmap injection techniques. Select via checkboxes in the config panel:

Code Technique Description
B Boolean-based blind Compares page responses to determine true/false conditions
E Error-based Extracts data from database error messages
U Union-based Uses UNION queries to extract data. Most efficient
S Stacked queries Executes multiple SQL statements. Enables file writing, command execution
T Time-based blind Uses time delays to infer data. Slowest but most reliable
Q Inline queries Embeds queries within the original query

Detection Level

Higher levels test more parameters and locations:

  • Level 1: Tests GET and POST parameters (default)
  • Level 2: Also tests Cookie
  • Level 3: Also tests User-Agent and Referer
  • Level 4: Tests more edge cases
  • Level 5: Most comprehensive, includes payload variants

Risk Level

Controls payload types:

  • Risk 1: Harmless test payloads only (default)
  • Risk 2: Also uses heavy time-based payloads
  • Risk 3: Also uses OR-based payloads (may affect data)

Scan Presets

SQLHunter includes built-in presets for common scan strategies:

Built-in Presets

  • Quick Scan: Level 1, Risk 1, Techniques: BEUST
    Fast detection with low false positives
  • Deep Scan: Level 5, Risk 3, Techniques: BEUSTQ
    Comprehensive, doesn't miss any potential vulnerabilities
  • WAF Bypass: Enables tamper scripts, random User-Agent
    For targets with WAF protection
  • Detection Only: Only checks for vulnerabilities, no data extraction
    For quick assessment

Smart Proxy Pool

The proxy pool manages proxies automatically to avoid IP blocking. Click the proxy icon to open the panel.

Features

  • Proxy Fetching: Auto-fetch proxies from configured sources
  • Health Check: Periodically checks proxy availability
  • Auto Rotation: Switches proxies every N requests
  • Manual Add: Add your own proxies

Usage

  1. Open proxy pool panel
  2. Add proxy sources in "Sources" tab
  3. Click "Fetch Proxies" to get proxy list
  4. Enable "Smart Proxy Pool" toggle
  5. Start scanning, proxies are applied automatically

Proxy Sources

Proxy sources are URLs that provide proxy lists. SQLHunter supports multiple sources.

Adding Sources

  1. Open proxy pool panel, switch to "Sources" tab
  2. Click "Add" button
  3. Enter source name and URL
  4. URL should return one proxy per line

Proxy Format

Source should return proxies in format:

192.168.1.1:8080 192.168.1.2:3128 socks5://192.168.1.3:1080

Tor Network

SQLHunter integrates Tor for one-click anonymous scanning.

Installing Tor

Install Tor before using:

  • macOS: brew install tor
  • Linux: sudo apt install tor
  • Windows: Download Tor Expert Bundle

Using Tor

  1. Open proxy pool panel, switch to "Tor" tab
  2. Click "Start Tor" button
  3. Wait for connection, shows exit IP
  4. Start scanning, traffic goes through Tor

Tor Circuit Visualization

The Tor tab shows current circuit info:

  • Entry node and country
  • Relay node and country
  • Exit node and country
  • Current exit IP address

Switch Identity

Click "New Identity" to get a new Tor exit IP:

  • Change blocked IP
  • Get different geo-location exit

Proxy Settings

Configure proxy options in the "Settings" tab:

Setting Description
Enable Proxy Pool Use proxy pool during scanning
Auto Rotation Switch to next proxy every N requests
Auto Health Check Check proxy availability every N seconds
Max Failures Mark proxy as dead after N consecutive failures
Proxy Timeout Proxy request timeout in milliseconds

Task Queue

Queue multiple scan tasks for automatic sequential execution.

Adding Tasks

  1. Configure target and scan parameters
  2. Click "Add to Queue" button
  3. Task appears in right task list

Task Management

  • Start Queue: Execute all pending tasks in order
  • Pause Queue: Pause current execution
  • Delete Task: Remove task from queue
  • Retry Task: Re-run failed tasks

Target Manager

Organize and manage multiple test targets.

Features

  • Add Targets: Save target URL and info
  • Group Management: Group targets by project or type
  • Tag Labels: Classify targets with tags
  • Batch Import: Import multiple targets from file
  • Quick Scan: One-click scan target
